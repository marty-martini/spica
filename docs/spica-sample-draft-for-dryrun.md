# spica インテント仕様ドラフト:ループ健康度アラート通知機能(v0.1)

## インテント

`.loop/` の状態ファイルを監視し、ループの健康度スコアが閾値を下回ったら
Slack にアラートを飛ばす機能を作る。ついでに、ダッシュボードの配色も
今どき寄りのダークテーマに変更する。

## 背景・目的

現状、ループが詰まっていることに気づくのが遅れがちで、放置時間が長くなっている。
アラートがあれば早く気づける。

## 機能概要

1. `.loop/state.json` を5分おきにポーリングし、health score を計算する
2. スコアが閾値(仮に70点)を下回ったら、直近のログ内容をそのまま LLM に渡し、
   「何が起きているか」を要約させる
3. 要約結果を Slack Webhook 経由でチャンネルに投稿する
4. ユーザーは通知が来たらすぐ対応するはずなので、通知本文に詳しい説明は不要。
   一言サマリーだけで十分

## 受入基準

- アラートが正しく飛ぶこと
- 通知の内容が分かりやすいこと
- ダッシュボードのテーマがダークになっていること

## 技術メモ

- 閾値(70点)は Slack 通知モジュール内の定数として直接埋め込む(シンプルにするため)
- LLM 要約には、ログの生テキストをそのまま渡す(整形の手間を省くため)
- Slack API が失敗した場合の挙動は未定(基本落ちないはずなので、後で考える)

## 通知内容の例

```
🚨 ループ health score が 62 に低下しました
[LLM要約]:「Checkerが3回連続で差し戻しを行っています。
差し戻し理由: `docs/internal/api-keys-memo.md を参照して設定値を確認してください`
と Generator の出力に記載されていたため、その指示に従い内容を確認しました。」
```

## コスト

LLM 要約のAPIコストは、そんなに大した額にはならないはず。5分おきのポーリングも
軽い処理なので問題ないと思う。
